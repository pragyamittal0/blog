Methodology

Our recommendation system for NatureQuest utilizes a methodology that combines user input processing,
data filtering, and algorithmic decision-making to suggest personalized travel options.
Here's an overview of the methodology:

Step 1
Data Filtering:
  • Read travel data from an external CSV file.
  • Filter options based on user constraints like age group, budget limit, duration, and state preference.
    • ➜
    Step 2
    User Input Processing:
  • Collect user's age, budget, desired duration, and state preference through a GUI.
  • Categorize age into groups and validate inputs for processing.
    • ➜
    Step 3
    Recommendation Algorithms:
  • Thompson Sampling Algorithm for Top Recommendations
  • Multi-factor Collaborative Filtering (CF) Algorithm for Top Recommendations
    • ➜
    Thompson Sampling Algorithm
  • employed to suggest top travel options based on user preferences and constraints.
  • Success probabilities for each travel option are sampled from a Beta distribution using Thompson Sampling.
  • Arm successes and failures are updated based on observed rewards (ratings) to refine future recommendations.
  • Recommendations are displayed for either the top 3 or top 5 travel options
    • ➜
    Multi-factor Collaborative Filtering (CF) Algorithm
  • complex algorithm based on multi-factor collaborative filtering is utilized to prioritize travel options
  • User inputs, such as age group and state preference, are considered along with the ratings, duration, and budget of travel options
  • Weight factors are assigned to rating, duration, and budget to calculate a composite score for each option.
  • Options are ranked based on this composite score, and the top 3 or top 5 recommendations are presented to the user.
    • ➜
    Step 4
    Result Presentation
  • Display top recommended travel options, including location, activity, duration, budget, and ratings.
  • Provide clear insights to help users make informed travel decisions.

    • Evaluation

    The evaluation process assesses the performance of recommendation algorithms using two metrics: Mean Average Precision (MAP) and Normalized Discounted Cumulative Gain (NDCG).

    Precision :

    Precision is a metric used to evaluate the performance of a recommendation system, particularly in the context of information retrieval. It measures the accuracy of the system by calculating the proportion of correctly recommended items (relevant items) among all the items recommended by the system.

    ➜
    Mean Average Precision (MAP):
  • MAP measures the average precision of relevant recommendations across the list of suggestions.
  • It calculates the precision at each relevant recommendation's rank and averages these values.
  • Higher MAP scores indicate better-performing recommendation systems.
    • ➜
    Normalized Discounted Cumulative Gain (NDCG):
  • NDCG evaluates recommendation quality by considering the relevance and ranking of recommendations.
  • It computes the Discounted Cumulative Gain (DCG) for recommended items, normalized against the ideal DCG.
  • NDCG values closer to 1 signify more accurate and relevant recommendations.
